# بحث المناقصات الممنوحة - Monaqasat Award Search

أداة بتدور تلقائيًا في **موقع مناقصات قطر** (monaqasat.mof.gov.qa) عن أي مناقصات ممنوحة سابقًا
لصنف معين، وتجيبلك كل تفاصيلها: الشركة الفايزة، القيمة المعتمدة، السجل التجاري، تاريخ المنح، الجهة، وغيرها.

الفكرة: بدل ما تفتح الموقع وتقلب صفحة صفحة (فيه أكتر من ١,١٠٠ صفحة!)، الأداة بتستخدم
خانة البحث الموجودة أصلاً في الموقع، وتجيبلك بس النتايج اللي فيها الصنف اللي كتبته،
وتحفظها في قاعدة بيانات محلية (SQLite) عشان تدور فيها تاني بسرعة من غير ما تنادي الموقع تاني.

---

## طريقة الاستخدام السريعة (على جهازك)

```bash
# 1. ثبّت المكتبات المطلوبة (مرة واحدة بس)
pip install -r requirements.txt

# 2. دور عن صنف
python search_tender.py "Medical Consumables"
```

هيطلعلك:
- كل المناقصات اللي فيها الصنف ده **بالظبط** (تطابق دقيق) مع كل التفاصيل والشركة الفايزة.
- ليستة بالنتائج التقريبية (لو حابب تشوفها كمان) في حالة مفيش تطابق دقيق كفاية.

كل نتيجة بتتحفظ تلقائيًا في `data/monaqasat.db` — المرة الجاية لو دورت على نفس الصنف أو صنف قريب،
النتائج المحفوظة هتظهر فورًا من غير انتظار.

---

## تشغيله من غير ما تحتاج جهاز أو سيرفر (مجانًا) — GitHub Actions

بما إنك مش عايز تدفع اشتراك لأي سيرفر، الحل المجاني هو **GitHub Actions**:

### الإعداد (مرة واحدة بس):
1. اعمل حساب GitHub مجاني لو مفيش عندك.
2. اعمل Repository جديد وارفعله الفولدر ده بالكامل.
3. من تبويب **Settings → Actions → General** في الريبو، تأكد إن صلاحية الكتابة مفعّلة
   (Workflow permissions → Read and write permissions) عشان الأداة تقدر تحفظ نتايجها في الريبو.

### الاستخدام اليومي (لما يجيلك RFQ):
1. روح لتبويب **Actions** في الريبو.
2. اختار workflow اسمه **"بحث عن صنف في المناقصات الممنوحة"**.
3. دوس **Run workflow**، اكتب اسم الصنف، ودوس Run.
4. بعد شوية (دقيقة أو اتنين) هتلاقي النتيجة في:
   - ملف `reports/last_search_result.txt` (اتحدث في الريبو تلقائيًا).
   - أو من جوه صفحة الـ Action نفسها (تبويب Logs).

ده بيشتغل من غير ما تسيب جهازك مفتوح، ومن غير أي اشتراك أو تكلفة.

## متابعة أصناف بشكل دوري (اختياري)

لو عندك أصناف بتتعامل معاها باستمرار وعايز تعرف أول ما تتمنح مناقصة جديدة فيها:

1. افتح ملف `watchlist.txt` وضيف كل صنف في سطر منفصل.
2. الـ workflow التاني (**"متابعة يومية لقائمة الأصناف"**) بيشتغل تلقائيًا كل يوم،
   يدور على كل الأصناف دي، ولو لقى مناقصات جديدة (مش موجودة قبل كده)، بيكتبها في
   `reports/new_matches.md` — تقدر تفتح الملف ده في أي وقت وتشوف إيه الجديد.
3. تقدر كمان تشغّله يدويًا وقت ما تحب من تبويب Actions.

## هيكل المشروع

```
monaqasat-search/
├── db.py                  # قاعدة البيانات (SQLite) - إنشاء الجداول والحفظ والبحث المحلي
├── scraper.py              # البحث الفعلي في موقع مناقصات قطر واستخراج البيانات
├── search_tender.py        # الأداة الرئيسية (تشغيل من الطرفية)
├── watchlist.py            # فحص دوري لقائمة أصناف محفوظة
├── watchlist.txt           # قائمة الأصناف اللي عايز تتابعها
├── requirements.txt
├── data/
│   └── monaqasat.db        # قاعدة البيانات المحلية (بتتبني تلقائيًا)
├── reports/
│   ├── last_search_result.txt
│   └── new_matches.md
└── .github/workflows/
    ├── manual-search.yml         # تشغيل بحث لحظي من GitHub
    └── scheduled-watchlist.yml   # فحص يومي تلقائي لقائمة المتابعة
```

## ملاحظات مهمة

- **البيانات كلها عامة ومنشورة أصلاً** على موقع مناقصات قطر الرسمي — الأداة دي بس بتوفر
  عليك وقت البحث اليدوي، مش بتوصل لحاجة مش متاحة للجميع.
- الأداة بتحط تأخير بسيط (1.5 ثانية) بين كل طلب والتاني احترامًا لسيرفر الموقع ومنعًا لأي حمل زيادة عليه.
- البحث بيدور بالظبط زي ما لو كنت فاتح الموقع بنفسك وكتبت في خانة "Tender Subject"،
  وبعد كده بيفلتر النتائج تاني عندنا عشان يطلعلك بس التطابق الدقيق فعلاً.
- لو الموقع غيّر شكل الصفحة بتاعته في المستقبل (تصميم جديد)، السكريبت ممكن يحتاج تعديل بسيط
  في `scraper.py` عشان يتطابق مع الشكل الجديد — تقدر ترجعلي وقتها وأظبطه.
